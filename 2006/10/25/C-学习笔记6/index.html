C#学习笔记6

发表于 2006-10-25

留言板的大致完工,极其简单的新闻系统的完工…倒是一个不错的标志,c# asp.net也就这样~ 前面那段是上手,那下面的学习计划就是要深入了 下面的计划是从最基础的开始,快速掠过,尽量不遗留知识点,重点的重点看,不是重点的不重点看(废话!),从以前略过的c++,c#基本语法,基础及各种数据结构各种基本对象开始慢慢看了,手头三大本书,花了2xx多rmb,加起来22xx页,无穷无尽,学海无边…………….. 呵呵

C#调用win32API 其实就导入Dll中的函数 public class MyWin32Wrapper { [DllImport(“User32.dll”)] static public extern void beep(int frequecy,int duration); } using语句,除了表示引用命名空间,还有就是定义一个对象使用的范围,在using结束之后对象自动释放 很好用,不像以前老是try{}finally{dispose()} 有个条件，能自动释放的类必须是实现IDisposible接口，这样才有dispose()，关于接口的概念暂时没研究，估计和delphi不会差别多少~ 大小写严格区分，这个已经吃过亏了 呵呵，一开始还很不习惯，不过在vs2005强大的自动完成功能下…… 运算符的重载，一个很有用的功能，也是个很难用好的功能 个人感觉，在没有特别的必要之前，少用为好，用的话，也尽量简单，比如类的相加相减完全可以通过一个方法来实现~ class Mile { static public Mile operator++(Mile mile) { } } 短路计算，在and or里面确定了值就不再算～没什么好说的，delphi早就有了 Int32.Phrse和Int32.TryPhase用来字符串转换到数字,还可以Try,返回T或者F ~ 字符串的处理 string.Format() 分组,””里面用{0},{1}表示,引号后面把变量list出来,自动对应,写SQL语句的时候很好用 其他还有日期格式化,时间,整数,浮点数格式化,等等,很多资料~ 字符串前面加一个@ 大部分转义问题都解决~ String有很多方法, EndWith() 是否以结束 IndexOf() IndexOfAny()子串的位置 StartWith()以开始 PadLeft()等,填充 Trim()方法,去除多余空格,Replace(“”,””)替换,str[]=str1.Split(“|”); 拆分,写论坛用得到, ToUpper等等大小写 StringBuilder 这个实际上是字符串流,能动态改变字符串长度(不用新建字符串),支持Append()追加方法,包括AppendFormat() 正则表达式 RegularExpression 这东西非常好用,功能异常丰富….在验证字符串的时候用的非常非常多(没词了),以后用到会详细研究,基本上常用的模板里都有~ 数组 不同于传统意义的数组,数组在这里也是一个对象 二维数组实际上是嵌套的对象~总之一切按照对象考虑问题就行了,所以也就出现了什么”锯齿数组” 实现ICollection接口的对象都算作集合 有Count GetEnumerator() CopyTo等方法 遍历最简单用for each CollType C in Collection{} 一些常用的数据结构都有实现: ArrayList Stack HashTable BitArray Queue…..用到再说 自定义集合 就是那个Address的例子 一般方法,定义一个类存放变量,及其方法 继承下来一个实现上面的方法,建立属性,其他方法等等 建立集合,从CollectionBase集成,重写必要的方法,将返回类型变为Address而非object 关于类,过些天详细研究….从UML开始~ ———– 没空了,还有文件操作和串行化操作….有空再来写 ——————— go on 文件的操作 基本上是通过Stream来进行 首先是对于文件本身的操作,有file类和FileInfo类,File类里面直接包含了文件操作(delete move copy)一类的方法,都是静态的,FileInfo需要构建实例,在对文件进行很多操作的时候,用FileInfo类是比较好的 Directory类,主要类似File类,都是一些对目录的操作 文件内容的操作,就是Stream了, 一般的文件,有FileStream 类似的还有MemoryStream StringReader StringWriter类 建立实例时一般会有相应参数比如文件名,打开方式(fileInfo.下面有) 读取流的数据用byte[] 然后Read()方法,每次可以读很多数据,具体在Read()方法里面有 如果是ASCII码,可以用System.Text.UTF8Encodeing类型的GetString方法获得 MemryStream和FileStream差不多 两个String的Stream是以stringbuilder为对象写入和读取数据,每次读取的都为一个char 还有StreamReader 和 StreamWriter两个类 可以以特定的编码方式从各种Stream中读取和写入文本,注意是文本 这两个类是从TextReader和TextWriter继承下来的…对于文本操作很方便~有ReadLine WriteLine OpenText()方法等等 FileSystemWatcher 一个很爽的类,作用居然是监视文件的变化…….. 设置path属性,path属性是需要监控的文件夹 包括子目录 Filter设置监视的文件扩展名 NotifyFilters设置文件更改的类型 缓冲区大小可以改变,防止大量更改时溢出… 有Changed deleted等事件,也有一些预先有的事件, // Begin watching. watcher.EnableRaisingEvents = true; 触发事件时,e提供相关信息 e.FullPath
e.ChangeType 建议新建一个FileSystemWatcher时候创建独立的进程~ 串行化.. 很好理解的东西,就是把类转化成可以保存成文件的格式 两种,一种是传统的串行化成二进制文件,一种是很爽的串行化成XML 串行化成XML有不少条件,比如很多地方必须为public等等 串行化成二进制文件,用binaryWriter的Serialize方法,提供一个FileStream和Value 串行成XML文件,用System.Xml.Serialization.XmlSerializer,创建时需要指定type 用typeof(ClassName) 然后用XmlSerializer的serializer方法,提供一个XmlTextWriter和一个object……. 大概就是这样,实现起来很爽,特别是Xml的串行化,说白了就是直接由类生成XML文件,把这个再和ajax联系起来写WebService……呵呵
