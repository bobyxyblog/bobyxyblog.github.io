C#学习笔记3

发表于 2006-10-11

稍微研究了下html控件,html控件，

前面提到过，<a <form之类的后面跟了runat=”server” 便会自动生成html控件

稍微看了一下，html组件使用很简单,能很好向前兼容,不过功能看似较单一,而web控件是asp.net新加的,所有控件都基于.net类库,最大的优点大概就是能够自定义组件…也就是可以继承或重新编写组件……

具体每个组件的用法暂时先不看,反正都大同小异,先单独看一下有一类叫验证控件的web组件,然后准备先把ado.net基础给看了

验证控件

requiredfieldvalidator 验证某个控件必须要存在值(值和初始值必须不同)

controltovaldate 指定需要验证的控件

errormessage 错误信息

initialvalue为初始值，

display none static dynamic 显示消息的方式,具体的msdn里有

comparevalidator 比较控件

大同小异 type属性指定数据类型

operator指定比较方式 equal notequal greaterthan lessthan datatypecheck一类 一共7个

valuetocompare…

rangevalidator 范围

用minimunvalue maximunvalue或者minimuncontrol之类的来确定最大最小值

最大最小值包括数字 字符 日期一类

type为类型

regularexpressionvalidator 表达式验证

用表达式来验证,功能很多 msdn有详细说明

validationexpression为正则表达式

所有验证控件,搜集到的信息,如果display设置的是none 都可以从validationsummary取得错误信息(不论是否已经显示)

有headertext showsummary displaymode属性 具体的msdn有 用到在说

还有从system.web.ui.usercontrol继承,编写用户控件

把webform变成用户控件

差点把这个漏了，其实aspx和ascx差别不大，把aspx文件中

都去掉，扩展名改成.ascx就行，，，ascx是用户控件，默认设置下只能通过本地页面调用
