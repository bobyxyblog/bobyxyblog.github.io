从吃的好开始~(1)

发表于 2005-10-21

从吃的好开始~ 校园卡系统设计 高三的中午的时间是宝贵的，可学校食堂的效率就那么的低下，长的吓人的队伍，凶神恶煞的“师傅”，质量实在不怎么地的菜……只能忍着了 却突然想到食堂的那套“校园卡”系统 无锡一中的“校园卡”其实也就只是饭卡，高一拿到卡时就感觉这东西质量不怎么样，估计是上个世纪九零年代初的东西了(系统还是foxbase 连同样差劲的中国石化都用foxpro…)。事实也证明，在这个学校里呆了两年多，饭卡换了无数张，而这只是我一个人，５块一张的卡算不上贵，不过想想学校在这上边大概也赚足了钱了吧~ 说说学校的饭卡系统 很明显，卡中存储的只是一个编号信息，这从加钱的过程可以看出来，那老师要的只是一个编号，我便可以扔下钱走人，同时，学校里应该有一台总的服务器储存了所有学生卡中的钱数（其实也就几千数据，数据量算不上大），刷卡时候能显示出现在卡中的钱数，而卡拿掉之后才去减钱。这样的话，食堂中每一台刷卡的终端应该都是联网的。 现在的问题不少，总结一下，大概有这么几点： １ 加钱很不方便，每次开学总务处都人满为患，相信每个一中的学生都是深有体会！ ２ 现在使用的磁卡性能比较差了，容易折断，磁条容易坏，而刷卡机由于有一定的年头了，工作起来更是想个老头子似的，什么都读不清楚。。。。每次刷卡的时候我总想，如果用菜票是不效率更高~ ３ 校园卡功能太单一，名曰“校园卡”却只能当饭卡，如果饭卡除了能在食堂使用之外，还能在小卖部，自动贩卖机使用，甚至用来交班费，交学费等等等等，若能实现这些功能，那一中的校园卡一定会成为一中的一个特色！ 所以很想给学校换一套饭卡系统，我还是重新来分析分析需求 系统需要完成的功能 １ 有些类似银行卡，但是只存钱，不取钱。（应该考虑在卡销毁时，也就是毕业时的退钱） ２ 校园消费，仅限于校园的消费，主要是食堂买饭，同时包括小卖部一类地方，但是自动贩卖机使用的实现有难度，主要是设备需要更新 ３ 身份标识，上面说到要实现班级班费管理等等功能，那么必须严格的遵循“一人一卡”。每张卡一个编号，每个编号对应一个人，这个编号应该统一编排，与班级，年级等无关（持卡人可能更换） …………未完待续 一些零散的想法，以后视情况加进来： １ 每个终端都是一台电脑肯定不实际，可能需要两种终端，一种是只会扣钱的那种，一种是能完成各种功能的终端（这个需要一台pc了），这种终端可以放在班级中，小卖部，办公室等等，可以加钱，查询帐务，扣钱（比如班级交钱，或者罚款 呵呵）等等 ２ 加钱一定要支持各种方式，只支持现金肯定太落后了 我想得到的方式有，现金，银行卡转帐，通过网络帐户（支付宝，paypal一类） 3 整个网络利用学校中局域网就行，但是如果能wi-fi 那岂不太爽~~~~ 4 既然是校园卡，不一定只能管理钱方面的事情，可以用来建立学生信用度 比如校内骑车扣分啊，，考试作弊处分啊，或者做了什么好事啊 都可以记在上面 呵呵….. 甚至是学生档案也可以部分的进到系统里边…. 这玩艺儿就算是教师节送给老师的礼物 呵呵

发表于 2005-10-21

接着上次的写系统需要完成的功能 １ 有些类似银行卡，但是只存钱，不取钱。（应该考虑在卡销毁时，也就是毕业时的退钱） ２ 校园消费，仅限于校园的消费，主要是食堂买饭，同时包括小卖部一类地方 ３ 身份标识，上面说到要实现班级班费管理等等功能，那么必须严格的遵循“一人一卡”。每张卡一个编号，每个编号对应一个人，这个编号应该统一编排，与班级，年级等无关（持卡人可能更换）4 终端上做的事情：根据卡查询关于这张卡的各类信息，若为班主任，则可以管理整个班级的事务，如交钱收钱等，若为总务处主任，则能完成相应任务５ 读卡器做的事情：就是扣钱了模块：１ 硬件： 食堂、小卖部等地方用读卡器：网上搜了下，总觉得比较好的就是现在公交用的那种卡，不用接触，反应快速，不过不清楚能不能实现饭卡那种输入一个数字扣除相应钱数的功能，这是个问题，在网上卖的那些机子大部分都得连着电脑用，否则就是那种很简单的读头。不过话又说回来，公共汽车上的那种应该就没什么问题，反面是有输入价钱功能的，还能语音提示，正面刷卡还有个显示，虽然没在网上找到有这样的读卡设备，但是我想这样的设备既然存在就应该能买得到，而且既然公交车上能够大规模的应用，那价钱就不应该太大。 终端设备上用的设备：这个是要连接电脑的，需要有强大的读写功能，设备倒是在网上看见了，价钱是3xx一台，我想全校每个教室一个应该也不是什么问题～～～要不就每个办公室一个 电脑：终端管理用的电脑学校应该是很好解决的，用不着很高配置的东西，学校淘汰下来的那些老赛扬应该就没问题～，服务器学校肯定有，这方面不是问题 网络：不清楚现在的食堂联网是怎么实现的，如果普通的以太网，那么换了系统肯定也没有任何问题，如果需要重新组建网络，那么也许就有些麻烦，利用现在的校园网络吧，只需要把网络通到食堂就行了，或者，在食堂里放一个服务器，通过wi-fi连接到附近的教学楼里，只是不知道距离够不够，呵呵软件： １ 读卡器只需要实现输入一个价钱然后刷卡扣钱就行了，很简单 ２ 终端电脑上面的是主头，得拿出一套高效率功能强大的学生校园卡管理系统，这个问题放在软件上面详谈 ３ 服务器软件，和上面一样，放在软件设计上面祥谈这个暂时就研究到这里，其实硬件方面读卡器厂商都会很好的解决的，以后主要研究软件的设计了~

我是从贵州吃出来的

发表于 2005-10-21

从小生长在大中国西南边的那个山洼洼里，独特的人，土地，还有气候，造就了当地特有的吃的文化。在这样的文化熏陶下，从呱呱坠地那一天开始算，一直到离开那块土地的好几个年头里，我时刻都是生活在浓香四溢的环境里。

今天不小心在网上看见了卖“贵州小吃”的，不禁想起了这些。

小时候最喜欢的是牛肉米粉，每天上学前一碗，为了体力——那个时候上学需要爬１０分钟的山——也为了给家里省事，以前早上８点上学，吃牛肉粉可以７点半才起床。

想起贵州的吃，和江苏的吃有很大个不同。这地方毕竟是江南，什么事情都是“吴侬软语”的感觉，小气得很。吃个包子，吃个糕点，都是小个小个的。这也就是为什么无锡拱北楼搞了个大碗面出来就会这么有名，所谓物以稀为贵。若是把拱北楼那大碗面送到贵州，也许就根本勾不起贵州人的食欲了，就如同他根本勾不起我的食欲一样。贵州人不像四川人，但凡吃的要讲个麻辣鲜香，极其讲究，也不像湖南人那样要的就是一个辣不怕，怕不辣，贵州人的食物要的是一个“满足生理需要”，解饿，顺带还得解个潮气，解给闷气。

牛肉粉就是这样的东西。

还有个印象很深的是折儿根。那玩艺儿取自鱼腥草之根。说是非贵州人吃不惯，而觉得难以下咽。确实这样，那是一股浓烈的药味，是一种让人无法忍受的气味。可是贵州人（比如我）闻起来，百闻不厌。这也许是天生的吧。

下午在家里研究delphi，却突然想起，书包里的那些东西和牛肉粉似乎有某种联系，而delphi，就好像是折儿根………………

文件拖动到程序里边（用API函数）

发表于 2005-10-21

程序得到来自外界的文件拖动列表

first use the dragacceptfiles function to register the window that need to accpet dropped files
dragacceptfiles(handles,true);

then handle the message wm_dropfiles
use function dragqueryfile to get the count of the dropped files(set $ffffffff as the second parameter) or to get the filename of from the filelist and put the name into a buffer..

at last,do not forget use dragfinish() to retrieve the handle…..but i don’t know what the function do here~~~~
maybe the function will end the message wm_dropfiles

the demo code:

procedure finishdropped(var msg: tmessage); message wm_dropfiles;

procedure tform1.finishdropped(var msg: tmessage);
var
hand: thandle;
buffer: array[0..255] of char;
i: integer;
count: integer;
files: tstringlist; //this is the file list
begin
files := tstringlist.create;
hand := thandle(msg.wparam);
count := dragqueryfile(hand, $ffffffff, buffer, 100);
for i := 0 to count - 1 do
begin
dragqueryfile(hand, i, buffer, 100);
files.add(strpas(buffer));
end;
dragfinish(msg.wparam);
memo1.lines:=files;
end;

档次

发表于 2005-10-21

以前玩电脑玩得不错，后来大了几岁，发现当时水平不过如此，现在档次不一样了

现在电脑玩得不错，跟前几年比，却发现没有质的变化

结果会的永远只是那些东西

清华的计算机系也没比我多学太多东西，但是他们是计算机高材生，我是业余爱好者，也是档次问题

就好像会打字的必不会打字的高一个档次，而100字/分的和50字/分的差不多一样

这个是量变和质变的关系

想起农村里几十年来出的第一个大学生，受到全乡人欢送，到了学校才发现自己成绩其实非常非常烂。。。真的很惨，最后有人自杀

差了好几个档次

高考650 要请客，高考600要请客，高考560，要请客，高考500 同样也请客

同样是考上大学本科的，档次却不一样

老师说的，自己考到了很理想的本科确认然被骂，这是家长要求与实际情况档次不同

有人考上复旦却闷闷不乐，因为清华落榜；有人考上个二本大学欢呼雀跃，因为分数本来只能上三本，档次不同也

档次高的人总是领先，而且很难追

就像期末考试的第二第三名次差一，分数差20 档次差一级

量的差别是好办的，档次的差别就不一样了

要想领先，比别人高一点是不够的，得比别人高一个档次

不过也许又是由一个档次也就是一点，但是不可能每一个档次都只相差一点

我还差几个档次？想来想去，还得上2个档次

提升档次知识，前途无量，只是难度不小
