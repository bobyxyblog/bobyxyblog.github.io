打印机
打印设备(print device):真实的打印机
打印机(printer):逻辑上的打印机
一个“打印机”可以对应多个“打印设备” 一个“打印设备”也可以对应多个“打印机”(优先级，权限可以不同),实现不同打印方式 打印服务器(printer server):连有打印机的pc
网络接口打印机（network-interface print device）

打印假脱机(printer spooler) 服务不启动，打印机无法工作
后台打印文件夹（缓存位置）

一个“打印机”对应多个“打印设备” 启用“打印池”,让“打印机”对应多个端口 (驱动程序必须相同 port本地远程都可以)
一个“打印设备”也可以对应多个“打印机” 创建多个“打印机”，端口指向同一个“打印设备” 不同优先级的多个”打印机”必须在一个机器上

共享时，对不同的客户端可以提供附加的驱动程序

打印权限：
打印：可以管理自己提交的文档
管理文档：同时可以别人管理的文档
管理打印机：对打印机所有操作

打印重定向：修改端口 在本地新建端口，指向远程打印设备 实现打印任务的转移

可能的错误： 1 权限不够 2 乱码（驱动问题） 3 server硬盘空间不够 4 打印stall乱码(重启print spooler服务)

网络打印机:添加本地打印机,新建tcp/ip端口

http打印,可以穿过路由器
服务器:共享打印机,安装iis
客户端:添加网络打印机 http://ip/printers/共享名/.printer

tcp:慢，确保正确性
udp:很快，依赖应用程序负责数据可用，在需要速度快或者很难获得回应时使用
icmp: ping
igmp:给一组计算机发消息
arp:负责ip和mac转化 1检查arp cache (若无对应mac)2发出广播，通过ip寻找目标mac 3所有收到的计算机add mac to cache 4 reply 5自己add mac to cache 6发送数据

application中，套接字：windows socket&netbt socket
windows socket使用dns解析 (host name)
netbt socket使用wins解析 (netbios name)

netbios
netbios名称： 不超过16字符 最后一位用16进制，标示服务

1 去服务器注册netbios name 2 查询 3关机，释放name

名称解析： 名称超过15字符或者包含”.”即转向dns解析 1使用cache 2使用专门netbios解析服务器(wins) 3通过广播查询 (lmhosts)
(可以通过wins去通过dns查找)
节点类型：b-node(广播) p-node(去问wins) m-node(先b后p) h-node(先p后b) (未指定wins默认b 指定wins默认h)
lmhosts文件： 本机host #pre 开机即载入cache

wins客户端开机时自动注册(原地址，目标地址，注册名 能跨网段) -> 收到响应 (含ttl) 到1/2ttl是刷新（重新注册）
netbios name 查询: net use x: \\ 即开始解析
释放：服务器把ttl设置为0

成为wins服务器：有固定ip 有wins服务

wins客户端访问非wins:在wins服务器添加wins数据库静态纪录 手动写上computername&ip
非wins需要wins服务器访问:通过广播找到一台wins proxy(静态ip)访问,转发wins server

解析另外网段的wins，通过wins数据库复制 复制伙伴(push/pull)：推（根据变化数目决定） 拉（根据时间决定）
保证高同步，用推；带宽有限，用拉 推荐：push/pull patern
先停止，再jetpack命令，压缩wins数据库

a b c类地址

e类:netid 1111开头，试验用地址，保留

特殊ip 0.0.0.0 缺省路由
255.255.255.255 全网广播
127...* 回路

或者通过子网掩码决定netid+hostid
子网掩码：确定主机是否则本网络内

开机时，通过ip+掩码,得到netid 通信时，通过对方ip和自己掩码得到netid 与自己相同，直接发，不相同，发给默认网关(路由器或其他路由器)

子网划分：借用hostid位来作为netid 255.255.192.0->11111111.11111111.11000000.00000000 14位可表示hostid 两位作为子网id 不能全0全1 有2^2-2=2个字网 即01**.**(64.1-127.254) 10**.**(128.1-191.254)**

cidr:(classless inter-domain routing)
不按照abc分类，通过子网掩码直接划分netid&hostid
131.104.64.0/18 18表示子网掩码中1的位数

同子网通信：3(原ip 目标ip) 2(原mac 目标mac)
不同子网通信：3(原ip 目标ip) 2(原mac 路由mac) -> 路由器收到后，取出目标ip mac改为目标mac -> 3(原ip 目标ip) 2(路由mac 目标mac）
在二层不停修改mac地址，不断发给下一个路由，最后发到目标~

3层，子网间传输；2层，子网内传输

路由器只关心netid

如果不连入internet 应用私有ip

网络的扩展
全部用hub:物理设备不同，但不同pc依然不能同时工作

网桥(bridge)：工作在第二层 能分析第二层报头 通过mac地址记录 端口 内网卡 两网卡端口一致，直接扔掉，端口不一致，则转发 ->网络分段,划分冲突域 广播帧直接转发

广播风暴:hub不能成环,bridge也不可以成环(广播帧) 路由其可以成环

路由器 router 工作在第三层,有ip协议
ip地址:networkid+hostid n.h结构
路由器根据netid进行路由, 所有广播帧会被丢掉,因此网络可以成环 能够自动选择一条合适的通路,在不同网段之间,数据包转发
不同协议间无法转发 路由表

tcp/ip ipx/spx可路由 netbeui,dlc不可路由

用bridge连接的网段,在ip层netid相同
用router连接的网段,netid一定不同

交换机:网桥或路由器~硬件转发 asic可以专门作为数据包转发使用 速度快! 能实现多路转发, 相当于多个网桥叠加,背板容量:同时转发的总速度

网关 gateway:大多数工作在第七层,协议的翻译,

广域网
1 pstn 电话线 用modem调制解调 56kmodem上传为33.6k
2 t1 美国常见的数字线路,1.544mbps ->分成24份 2464k 中国:e1 2.048mbps 分成3264k
3 isdn 仍使用电话线,本地交换机上使用数字电话网卡 本地需要nt(net terminal) device接数字电话机, 用ta(terminal adaptor)接到pc或者模拟电话上 2b+d (1*64k+16k)b:数据通道 d:控制通道
4 分组交换(packet swiching) x.25 模拟连接
帧中继(frame relay):数字线路上的分组交换网络 数据包比较大,可变 pvc技术
5 atm异步传输模式 发送固定大小数据包(5+48字节) 速度快155m 622m
6 adsl: 利用4khz-1mhz 高速~ 虚拟拨号
7 cable modom:通过电视信号

tcp/ip协议
适合c/s结构,
1 网络接口层 network interface
2 internet ip协议 internet protocol 大多数数据包最终转到ip协议 arp协议->把ip地质翻译成mac地址 icmp(internet control message prtocol)传输控制协议 igmp负责多目广播
3 传输层transport tcp udp 工作在传输层，基于ip协议工作
4 应用层application: telnet远程登录协议 ftp telnet ftp smtp基于tcp dns,rip(交换路由表),snmp(简单网络管理协议，检查网络状况)基于udp nntp(network news transport protocol新闻组)

apipa 自动专用ip寻址 没有设置ip且无dhcp服务器,默认169.254.. 遇冲突自动产生10次ip

传输层tcp协议：保证数据传输正确 udp不建立连接，不保证数据传输正确

基带传输(baseband):离散的数字信号 占用全部带宽，同时只能传送一个信号，分时传输
宽带传输(broadband):模拟信号

网卡：并->串
i/o地址 类似0x300 表示 系统调用设备的地址
中断号irq：设备打断cpu的标识 0-15
pci总线一般可以自动识别irq，能共享irq
mac地址 6字节 前3字节->厂商 后3字节->序列号 全世界唯一

数据包:mac地址标示,或广播地址
1直接帧 p2p通讯 目标地址为目标网卡mac地址
2广播帧 目标地址为广播地址 越少越好
3多目广播帧 对选定的目标发送信息

访问机制:如何把数据发到网线上
主要机制:
csma/cd,侦测到网线上有数据包,则等待,否则发送一个包,冲突则停止发送
高负荷时冲突增多,效率下降,实际带宽降低
令牌传递(toker passing):拥有令牌的主机,则能够发送数据(可以避免冲突) 发送完之后把令牌交给下一台 大负荷的时候效率不下降

osi模型^_^ 七层结构 每一层帮助上层完成工作
应用层:完成用户操作,包括各种协议(ftp http smtp dns…)
表达层，会话层
上三层面向用户
下四层提供可靠p2p数据传输
传输层：完成数据包的可靠传输 负责分段重组
网络层：传送有大小限制的数据包 负责如何在子网间传递数据，寻址
数据链路层：不但加报头，而且加报尾（带效验，确保正确性）负责子网内的传输
packet->frame(二层以下)
物理层：负责网线传输

modem：和网卡一样工作第二层，通过电话线来传送数据

以太网：第一第二层的实施 csma/cd机制

itu,ccitt电信组织~

ieee802模型
第一层第二层的细化
二层分为：
逻辑链路控制层: (llc) 负责差错和流量控制
媒体访问子层:(mac) 负责访问控制

以太网：是总线型 基带 csma/cd 802.3
10base2 细缆 bnct 185m
10base5 粗缆
10baset 双绞线 最短2.5m 最长100m
100baset4:能在3类线实现100m网络 已不用

100basefx:光纤传输

5-4-3规范，两台主机，最多有5条网线，4个中继器，（对于细缆最多3根网线可以接机器）只适合10m

速度 4mbps&16mbps
